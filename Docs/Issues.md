# GymRush — 이슈 브레이크다운 (순차 버전)

기획서(`모바일 아케이드 아이들 게임_ 헬스장 타이쿤.pdf`)와 현재 저장소(`2eesh/GymRush`) 코드를 대조해서 정리한 작업 목록입니다.

**이 문서는 위에서 아래로 순서대로 진행하면 되는 하나의 번호 목록입니다.**
각 이슈는 그 자체로 GitHub 이슈 1개가 되도록 최대한 작게 쪼갰고, "선행" 항목에 적힌 번호의 이슈가 끝나야 시작할 수 있습니다. 선행이 "없음"인 이슈는 지금 바로 시작 가능합니다.

> `#N` = 이 문서 안에서의 이슈 번호입니다. `gh issue create`로 이 순서 그대로 하나씩 등록하면 실제 GitHub 이슈 번호도 거의 동일하게 맞아떨어집니다 (다른 이슈가 중간에 안 생긴다는 가정 하에).

> **게이지 원칙**: 진행률 게이지는 플레이어나 NPC/AI 캐릭터를 따라다니지 않습니다. 게이지는 항상 **가이드 마커(카운터 등) 또는 기구 프리팹 자체에 고정으로 붙어있는 자식 오브젝트**입니다. 그래서 "대상 추적/Billboard 따라다니기" 로직은 이 문서에 없습니다.

## 구간 2. 가이드 마커 & 상호작용 공용 시스템 — #6~#12

카운터 체크인, 기구 청소 등 여러 곳에서 재사용할 공용 컴포넌트입니다. 게이지는 플레이어/AI를 따라다니지 않고, **마커나 기구에 고정으로 붙는 자식 오브젝트**입니다.

### #6. GuideMarker 컴포넌트 기본 구조
- **선행**: 없음
- **설명**: 프리팹 자식 오브젝트로 붙는 컴포넌트. 정지 지점 좌표를 갖고, `Show()`/`Hide()` API만 우선 구현 (트리거/타이머 없이).
- **완료 조건**: 임의의 오브젝트에 `GuideMarker`를 붙이고 코드로 보이기/숨기기가 된다.

### #7. IInteractable 인터페이스 정의
- **선행**: 없음
- **설명**: `OnEnter()`, `OnProgress(float ratio)`, `OnComplete()`, `OnCancel()` 콜백을 갖는 인터페이스만 정의 (구현체 없음).
- **완료 조건**: 인터페이스가 컴파일되고, 빈 구현 클래스를 하나 만들어 붙여볼 수 있다.

### #8. 진행률 게이지 UI 프리팹 (재사용 컴포넌트)
- **선행**: 없음
- **설명**: 마커나 기구 프리팹의 자식으로 고정 배치되는 Slider UI. `SetProgress(0~1)` API 제공. 플레이어/AI를 따라다니는 로직은 없음 — 부모 오브젝트(마커/기구)에 이미 자식으로 붙어있으므로 위치는 자동으로 따라감.
- **완료 조건**: 이 프리팹을 마커나 기구 밑에 자식으로 넣으면 그 자리에 게이지가 고정으로 보이고, 코드로 진행률을 0→1로 바꾸면 차오른다.

### #9. GuideMarker에 Trigger Collider 부착
- **선행**: #6
- **설명**: 마커의 정지 지점에 Trigger Collider를 붙여 플레이어 진입/이탈을 감지만 하도록 구현 (로그 출력 정도로 확인).
- **완료 조건**: 플레이어가 마커 콜라이더에 들어가고 나갈 때 진입/이탈 이벤트가 감지된다.

### #10. 정지 판정 + 상호작용 타이머 진행/리셋 로직
- **선행**: #7, #9
- **설명**: 플레이어가 마커 영역 안에 있고 이동속도가 0일 때만 타이머가 진행, 움직이면 타이머 리셋. `IInteractable.OnProgress()` 호출.
- **완료 조건**: 마커 영역에서 멈추면 타이머 값이 오르고, 움직이면 즉시 0으로 리셋된다.

### #11. 상호작용 타이머 ↔ 마커 게이지 연동
- **선행**: #8, #10
- **설명**: 마커 자식으로 붙은 #8 게이지에 #10의 진행률을 전달.
- **완료 조건**: 마커 영역에서 멈춰 있으면 그 마커에 고정된 게이지가 실시간으로 차오른다.

### #12. 상호작용 완료 처리 + 마커 숨김
- **선행**: #10, #6
- **설명**: 타이머가 100%에 도달하면 `IInteractable.OnComplete()` 호출 후 `GuideMarker.Hide()` 실행.
- **완료 조건**: 게이지가 다 차면 완료 콜백이 호출되고 마커가 사라진다. (이 시점까지는 `OnComplete`가 아무 동작도 안 해도 됨 — 실제 동작은 이후 이슈에서 연결)

### #38. 비어있는 기구 탐색 로직
- **선행**: #28, #37
- **설명**: 필드에서 `Empty(0)` 상태인 기구를 찾는 함수.
- **완료 조건**: 함수 호출 시 현재 비어있는 기구 중 하나의 좌표를 반환한다.

### #39. 결제 완료 손님 → 기구로 이동
- **선행**: #36, #38
- **완료 조건**: 결제를 마친 손님이 비어있는 기구를 스스로 찾아 이동해 도착한다.

### #40. 기구 도착 → 운동 상태 + 기구 부착 게이지
- **선행**: #39, #8
- **설명**: 기구 프리팹에 #8 게이지를 자식으로 부착해두고, 손님이 도착하면 운동 상태로 전환하며 그 기구의 게이지가 차오르도록 진행률을 갱신. 게이지는 손님을 따라다니지 않고 기구에 고정.
- **완료 조건**: 손님이 기구에 도착하면 운동 상태로 전환되고, 그 기구에 고정된 게이지가 나타나 차오른다.

### #41. 운동 타이머 완료 → 퇴장 처리
- **선행**: #40
- **완료 조건**: 게이지가 다 차면 손님이 기구에서 일어나 출구로 이동 후 사라진다.

### #42. PT존 이용 손님 지폐 드롭
- **선행**: #41, #13
- **완료 조건**: PT존을 이용한 손님이 사라지기 직전 바닥에 지폐 더미가 생성된다.

### #43. 대기 불만족 게이지
- **선행**: #36, #38
- **설명**: 대기열 정체 또는 빈 기구 없음 상태가 길어지면 차오르는 게이지. (이 게이지도 손님 캐릭터를 따라다니는 UI가 아니라 손님 오브젝트 자체에 고정 부착)
- **완료 조건**: 손님을 오래 대기시키면 불만족 게이지가 차오르는 게 보인다.

### #44. 불만족 100% → 분노 퇴장
- **선행**: #43
- **완료 조건**: 게이지가 100%가 되면 화남 이모티콘이 뜨고 결제 없이 즉시 퇴장한다.
